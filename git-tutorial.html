1 Git 核心概念

代码时光机

记录每一次改变

Git 是现代软件开发的核心工具，它就像代码的"时光机"，能记住你每次的代码修改。当你需要时，可以随时回到之前的版本。

简单说：Git能帮你记录代码的每一次变化，防止代码丢失，还能让多人协作写代码变得轻松。

Git 超能力

  • 多人协作不冲突
  • 离线也能工作
  • 完整修改历史
  • 强大的分支功能

核心概念

  • 仓库：代码的家
  • 提交：保存一次修改
  • 分支：代码的平行宇宙
  • 合并：整合不同分支

2 安装与配置

下载安装 Git

访问 Git 官网下载对应操作系统的安装包：

Windows 下载地址
https://git-scm.com/download/win
macOS 下载地址
https://git-scm.com/download/mac
Linux (Ubuntu/Debian)
sudo apt-get install git

安装提示： Windows安装时，勾选"Add Git to PATH"选项（这样可以在任何地方使用Git命令）。

初始配置

安装完成后，需要告诉Git你是谁：

设置你的名字
git config --global user.name "你的名字"
设置你的邮箱
git config --global user.email "你的邮箱@example.com"
查看配置信息
git config -l

重要： 名字和邮箱要真实，这样别人才能知道是谁修改了代码。

3 核心命令

仓库操作

初始化新仓库
git init
克隆远程仓库
git clone https://github.com/用户名/仓库名.git

查看与比较

查看仓库状态
git status
查看提交历史
git log --oneline
查看文件差异
git diff 文件名

提交修改

添加单个文件
git add 文件名
添加所有文件
git add .
提交修改
git commit -m "修改说明"

撤销操作

撤销文件修改
git checkout -- 文件名
撤销暂存文件
git reset HEAD 文件名
修改上次提交
git commit --amend -m "新说明"

4 分支管理

分支就像代码的"平行宇宙"，你可以在新分支上修改代码，不会影响主分支。改好后可以合并回主分支。

创建与切换

创建新分支
git branch 新分支名
切换分支
git checkout 分支名
创建并切换
git checkout -b 新分支名

合并操作

合并分支
git merge 分支名
变基操作
git rebase 目标分支
解决冲突后继续变基
git rebase --continue

删除分支

删除已合并分支
git branch -d 分支名
强制删除分支
git branch -D 分支名
删除远程分支
git push origin --delete 分支名

5 远程协作

远程仓库就像代码的"网盘"，可以把代码存到网上（如GitHub），方便团队共享和备份。

上传代码

添加远程仓库
git remote add origin 仓库地址
第一次推送
git push -u origin main
后续推送
git push

下载代码

获取最新代码
git fetch
拉取并合并
git pull
拉取特定分支
git pull origin 分支名
